## Member Training

### Module 1: Getting Started

#### Lesson 1: First-Time Login

**Learning Objectives:**
- Access the KMainCMS system
- Complete initial setup
- Navigate the dashboard

**Step-by-Step Guide:**

1. **Access the System**
   - Open your web browser
   - Go to your church's KMainCMS URL
   - Enter your email and password
   - Click "Sign In"

2. **Complete Your Profile**
   - Click on your profile icon
   - Select "Edit Profile"
   - Fill in your personal information
   - Upload a profile picture
   - Click "Save"

3. **Tour the Dashboard**
   - Review the welcome message
   - Explore the quick statistics
   - Check upcoming events
   - Read recent announcements

**Practice Exercise:**
- Log in to the system
- Complete your profile setup
- Navigate to each main menu item
- Log out and log back in

#### Lesson 2: Understanding the Interface

**Learning Objectives:**
- Understand the main navigation
- Use the search function
- Customize your dashboard

**Step-by-Step Guide:**

1. **Main Navigation**
   - Dashboard - Your home page
   - Members - View member directory
   - Events - Church events and activities
   - Announcements - Church news and updates
   - Giving - Donations and financial contributions
   - Groups - Small groups and ministries
   - Documents - Church documents and resources
   - Gallery - Photos and media

2. **Using Search**
   - Click the search icon
   - Type what you're looking for
   - Review search results
   - Click to navigate

3. **Customizing Dashboard**
   - Click "Customize" on dashboard
   - Add/remove widgets
   - Arrange widgets by dragging
   - Save your layout

**Practice Exercise:**
- Use search to find a specific member
- Customize your dashboard layout
- Add and remove widgets
- Save your preferred layout

## Developer Training

### Module 1: Development Setup

#### Lesson 1: Local Development Environment

**Learning Objectives:**
- Set up development environment
- Install dependencies
- Run development servers

**Step-by-Step Guide:**

1. **Prerequisites**
   - Install Node.js 18+
   - Install PostgreSQL 15+
   - Install Redis 7+
   - Install Git

2. **Project Setup**
   ```bash
   git clone <repository-url>
   cd KMainCMS
   
   # Backend
   cd backend
   npm install
   
   # Frontend
   cd ../frontend
   npm install
   ```

3. **Environment Setup**
   ```bash
   cp .env.example .env
   # Edit .env with your settings
   ```

4. **Start Development**
   ```bash
   # Backend
   cd backend
   npm run dev
   
   # Frontend (new terminal)
   cd frontend
   npm run dev
   ```

**Practice Exercise:**
- Complete development setup
- Start both servers
- Access the application
- Make a small change and test

#### Lesson 2: Understanding the Codebase

**Learning Objectives:**
- Understand project structure
- Navigate the codebase
- Understand architecture

**Step-by-Step Guide:**

1. **Project Structure**
   - Review directory structure
   - Understand module organization
   - Identify key files

2. **Backend Architecture**
   - Routes → Controllers → Services → Repositories
   - Review each layer
   - Understand data flow

3. **Frontend Architecture**
   - Pages → Components → Contexts → Services
   - Review each layer
   - Understand state management

**Practice Exercise:**
- Explore project structure
- Trace a request through backend
- Trace a user action through frontend
- Identify key files for a feature

### Module 2: API Development

#### Lesson 1: Creating API Endpoints

**Learning Objectives:**
- Create new API endpoint
- Implement business logic
- Test the endpoint

**Step-by-Step Guide:**

1. **Create Repository**
   - Create file in `repositories/`
   - Implement data access methods
   - Add error handling

2. **Create Service**
   - Create file in `services/`
   - Implement business logic
   - Call repository methods

3. **Create Controller**
   - Create file in `controllers/`
   - Implement request handlers
   - Call service methods

4. **Create Routes**
   - Create file in `routes/`
   - Define API endpoints
   - Add middleware

5. **Register Routes**
   - Import in `server.js`
   - Mount routes
   - Test endpoints

**Practice Exercise:**
- Create a simple API endpoint
- Implement all layers
- Test with Postman/curl
- Document the endpoint

#### Lesson 2: Frontend Integration

**Learning Objectives:**

**Step-by-Step Guide:**
